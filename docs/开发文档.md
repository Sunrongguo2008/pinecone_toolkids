# Toolkids 开发文档

> 本文件面向“以后维护/扩展这套代码的人”（也包括把它交回给 AI 继续写的场景）。
> 需求与兼容性细节见本目录的 [`需求.md`](./需求.md)。

## 1. 项目简介

Toolkids 是一个“便携式工具箱”（类图吧工具箱）：

- 左侧分类、右侧软件图标网格；
- 软件“便携化”运行：启动前还原、退出后备份并清理系统注册表/目录（本质是**备份/还原**，不是运行期真隔离）；
- 轻量级沙盒扫描：快照对比生成配置。

目标环境：**Windows 7 SP1 x64 ~ Windows 11 + WinPE**。为兼容性采用扁平外观，不使用 Win11 云母/亚克力等需要 DWM 的特效。

## 2. 技术栈

| 项 | 选择 | 说明 |
|---|---|---|
| 运行时 | **.NET 6**（`net6.0-windows`） | 最后一个支持 Win7 SP1 的 .NET，且能自包含 |
| UI | **WinForms** | GDI 绘制，不依赖 DWM，PE 里最稳 |
| 发布 | 自包含单文件，**仅 x64** | 免装运行时；详见第 3 节 |
| JSON | **Newtonsoft.Json** | 唯一第三方依赖 |
| 权限 | `app.manifest` `requireAdministrator` | PE 下本就是管理员 |

## 3. 如何编译运行

### 前置

- 安装 .NET SDK（本机已装 8.0.422，路径 `C:\Program Files\dotnet`）。
- SDK 用 8.x 即可，项目目标框架是 `net6.0-windows`，发布时会自动拉取 net6.0 运行时包。

### 命令

在 `src/Toolkids/` 目录下：

```bash
# 编译验证（快速，不打包运行时）
dotnet build -c Debug

# 发布：自包含 + 单文件 + x64，输出到 dist/Toolkids/
# 发布：自包含 + 单文件 + 压缩 + x64，输出到 dist/Toolkids/
dotnet publish -c Release -r win-x64 --self-contained true \
  -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true \
  -p:EnableCompressionInSingleFile=true \
  -o ../../dist/Toolkids
```

产物：`dist/Toolkids/Toolkids.exe`（压缩后约 61MB；`.pdb` 是调试符号，分发时可不带）。

> 压缩会让首次启动稍慢（解压到 %TEMP%）；若覆盖发布时报“资源占用/Device busy”，是旧 exe 正在运行，先关掉它，或先发布到新目录再替换。

### 注意

- `.bat` 脚本在中文 Windows 上必须 **ASCII + CRLF**，否则 cmd 会乱解析；本项目不提供 `.bat`，直接用上面的 `dotnet` 命令。
- 单文件版首次运行会把原生库释放到 `%TEMP%`；PE 下 `%TEMP%`（X: 内存盘）可写，没问题。若遇到只读环境导致释放失败，改用非单文件发布（去掉 `PublishSingleFile`）。

## 5. 架构与分层

三层 + 组合根，依赖方向单向向下，**UI 不直接碰文件/注册表**：

```
Program (组合根)
   │  new AppServices(...)  ── 一次性构造所有服务
   ▼
UI 层 (MainForm / Dialogs)        只做界面与交互
   │  调用
   ▼
Services 层                       全部业务逻辑
   │  操作
   ▼
Models 层 (POCO)  +  文件系统 / 注册表
```

要点：

- **手动依赖注入**：所有服务在 `AppServices` 里集中构造，通过构造函数传进窗体；不使用全局静态单例（`AppTheme.Current` 仅用于让对话框取当前配色，是唯一例外）。
- **窗体很薄**：`MainForm` 的事件处理只做“取输入 → 调服务 → 刷新界面 → 出错提示”，不写业务逻辑。
- **Models 是纯数据**：只有属性，没有方法/IO，方便 JSON 序列化与测试。
- **错误处理**：服务层抛异常，UI 层 `try/catch` 给中文提示并记日志；`Program` 里还有一道全局兜底。

## 6. 服务职责速查

| 服务 | 职责 | 不负责 |
|---|---|---|
| `AppPaths` | 解析根目录/可写目录/日志路径，探测可写性 | 读写配置 |
| `PeEnvironment` | 判断是否 WinPE | 其它 |
| `ConfigStore` | 序列化读写两种配置文件 | 决定业务规则 |
| `ToolRepository` | 扫描/增删 Data 下的软件文件夹 | 启动软件 |
| `IconService` | 取图标（带缓存） | 业务 |
| `AppLauncher` | 启动进程（阶段2 接沙盒） | 配置读写 |

## 7. 代码规范

- 一个文件一个主要类型；紧密相关的小类型（如同属一个 JSON 文件的几个类）可放同一文件。
- 命名：类型/公开成员 PascalCase，私有字段 `_camelCase`，常量 PascalCase。
- 启用 **可空引用类型**（`<Nullable>enable</Nullable>`）：可能为空的用 `?`，不为空的保证初始化。
- 公开类型/方法写简洁中文 XML 注释（`///`）。
- 注释解释“为什么”，不复述“做了什么”。
- UI 字符串集中在用到处即可（暂不做多语言）；用户可见错误一律中文。

## 8. 配置文件结构

字段定义见 `需求.md`。要点：

- `kidconfig.conf`：主题、布局、Data 目录名、分类列表、行为开关、扫描范围。
- `toolconfig.json`：名称、简介、图标、`launch`（exe/args/workingDir/runAsAdmin/waitMode）、`sandboxRules`（registry/files，支持 `%AppData%` 等变量）。
- 枚举（如 `waitMode`）以字符串形式序列化（`StringEnumConverter`）。

## 9. 阶段路线图

- **阶段1（已完成）**：主界面、分类与软件增删改、配置读写、启动软件、主题、PE 探测与可写回退、日志、异常兜底。
- **阶段2（已完成）**：沙盒备份/还原，代码集中在 `Services/Sandbox/`：
  - `SandboxEngine` 编排：启动前还原 → 运行并等待 → 退出后备份并清理；
  - `ConflictDialog` 覆盖确认（受 `ConfirmConflictBeforeRestore` 控制）；
  - `ProcessWaiter` 按 `waitMode` 等待，`Tree` 用 Job Object 跟踪整棵进程树；
  - `SandboxJournal` 事务日志，崩溃/断电后下次运行自动“恢复清理”；
  - `RegistryHelper`(reg.exe 导入导出，64 位视图) + `FileOps`(复制/带重试删除) + `SandboxStorage`(Backup 布局) + `EnvPaths`(变量展开/安全名)；
  - UI 经 `ISandboxInteraction` 回调，引擎在后台线程跑，不卡界面；`SkipSandboxInPE` 为真且在 PE 时整体跳过。
- **阶段3（已完成）**：沙盒扫描（快照对比法），`Services/Sandbox/`：
  - `Snapshotter`：在 `SnapshotScope` 范围内拍注册表/文件快照并对比，只取“新增”且折叠到最顶层项；
  - `ScanEngine`：拍快照 → 启动并等待退出 → 再拍 → 对比 → 勾选 → 写回 `SandboxRules`；
  - `ScanResultDialog` 勾选界面；文件路径经 `EnvPaths.ToPortable` 还原成 `%AppData%` 等变量。
  - 配套测试程序见 `testapp/`（写 `HKCU\Software\ToolkidsTest` 与 `%AppData%\ToolkidsTest`）。
- **阶段4**：PE 适配细节、单文件压缩瘦身、图标大图提取、对话框深色化等收尾。

## 10. 已知限制（当前）

- 图标用 `Icon.ExtractAssociatedIcon`（32px），大网格下略糊，阶段4 改大图标提取。
- 注册表：**HKLM / HKCR 的规则自动同时处理 32 与 64 两个视图**（32 位程序落在 WOW6432Node 的项也覆盖到，备份文件分 `.64.reg` / `.32.reg`）；HKCU 等不受 WOW64 重定向影响，走单视图。扫描到的 WOW6432Node 路径会规整成干净的逻辑路径。
- 软件运行期间是真写进真实系统的（备份/还原语义，非运行期隔离）；同一软件不允许并发启动（`_running` 拦截）。
- 个别系统对话框（文件选择框等）在深色主题下样式有限，属 WinForms 固有限制（ListView 列头已自绘为深色，列表视图最后一列自动填满）。
